🔧 Gulp Workflow & SASS Integration

Development Workflow (src/ folder)

  • index.html - Your HTML structure with design elements
  • index.scss - Component-specific SASS styles
  • _variables.scss - Local SASS variables (colors, fonts, spacing)
  • _mixins.scss - Reusable SASS patterns

Build Process (gulpfile.js)

// Example gulpfile.js structure
const gulp = require('gulp');
const sass = require('gulp-sass');
const autoprefixer = require('gulp-autoprefixer');
const cssnano = require('gulp-cssnano');

// SASS compilation
gulp.task('sass', () => {
  return gulp.src('src/**/*.scss')
    .pipe(sass().on('error', sass.logError))
    .pipe(autoprefixer())
    .pipe(gulp.dest('css/'));
});

// Watch for changes
gulp.task('watch', () => {
  gulp.watch('src/**/*.scss', gulp.series('sass'));
  gulp.watch('src/**/*.html', gulp.series('html'));
});

🎯 Complete Route Creation Process

Step 1: Create Controller Action

Create or modify a controller in src/Core/Controllers/ directory.

Example: HomeController.php 
<?php
namespace App\Core\Controllers;

use App\Core\Http\Controller;

class HomeController extends Controller
{
    public function index()
    {
        // Data passed here will be available in template as $this->parameters
        $this->parameters = [
            'title' => 'Welcome to My App',
            'description' => 'A beautiful landing page',
            'posts' => [
                ['title' => 'First Post', 'content' => 'Content here'],
                ['title' => 'Second Post', 'content' => 'More content']
            ]
        ];
        
        // Return view name (corresponds to template file)
        return 'home/index';
    }
    
    public function about()
    {
        $this->parameters = [
            'title' => 'About Us',
            'team' => ['John', 'Jane', 'Bob']
        ];
        
        return 'home/about';
    }
}

Step 2: Create Template File (.tpl)

Create a .tpl file in src/Core/templates/{controller}/{action}.tpl

Example: src/Core/templates/home/index.tpl 

                  #app | prepend = <?php $data = $this->parameters;?>
              
              h1[data-v-title] | innerText = <?php echo $data['title']; ?>
              h3[data-v-description] | innerText = <?php echo $data['description']; ?>
              p[data-v-intro] | innerText = <?php echo $data['intro']; ?>
              
              #instructions | prepend = <?php foreach($data['instructions'] as $instruction) { ?>
                  li[data-v-instruction] | innerHTML = <?php echo $instruction; ?>
              #instructions | append = <?php }?>
              
              a[data-v-action] | innerText = <?php echo $data['button_text']; ?>

Step 3: Create HTML Source File

Create the HTML file in themes/landing/src/{controller}/{action}.html

Example: themes/landing/src/home/index.html 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Home Page</title>
</head>
<body>
    <div id="app">
        <!-- This is where your .tpl content will be injected -->
        <!-- The #app selector tells the system where to append content -->
    </div>
</body>
</html>

Step 4: Auto-Generation Process

The system automatically:

  1. Reads from themes/landing/src/ folder
  2. Processes the .tpl template
  3. Generates final HTML in themes/landing/{controller}/ folder
  4. Injects processed content into the HTML file

🔧 Key Concepts

Template Syntax

  • #app | append = - Tells the system to append content to #app element
  • $this->parameters - Contains all data passed from controller
  • Heredoc syntax - Use <<<HTML for clean HTML writing
  • Variable interpolation - Use {$variable} inside heredoc

Data Flow 

Controller → Template (.tpl) → HTML Source → Generated HTML
    ↓              ↓              ↓              ↓
Parameters → Process Data → Inject Content → Final Output

🎨 Template Examples

Simple Data Display 

#app | append = <?php
$data = $this->parameters;
echo "<h1>{$data['title']}</h1>";
echo "<p>{$data['description']}</p>";
?>

Loop Through Data 

#app | append = <?php
$data = $this->parameters;
echo "<ul>";
foreach ($data['users'] as $user) {
    echo "<li>{$user['name']} - {$user['email']}</li>";
}
echo "</ul>";
?>

Conditional Rendering 

#app | append = <?php
$data = $this->parameters;
if (!empty($data['posts'])) {
    echo "<div class='posts'>";
    foreach ($data['posts'] as $post) {
        echo "<article><h2>{$post['title']}</h2></article>";
    }
    echo "</div>";
} else {
    echo "<p>No posts available.</p>";
}
?>

⚠️ Important Notes

  1. Always use $this->parameters to access controller data in templates
  2. Template files (.tpl) are processed by the Vtpl system
  3. HTML source files go in themes/landing/src/ folder
  4. Generated files appear in themes/landing/{controller}/ folder
  5. Use #app selector in HTML source for content injection
  6. Template syntax must be valid PHP code

🔍 Troubleshooting

Common Issues

  • Template not found: Check file path and naming convention
  • Data not available: Ensure $this->parameters is set in controller
  • HTML not generated: Verify HTML source file exists in correct location
  • Syntax errors: Check PHP syntax in .tpl files

Debug Tips 

// In your template, add debugging
#app | append = <?php
var_dump($this->parameters);
echo "<pre>" . print_r($this->parameters, true) . "</pre>";
?>
